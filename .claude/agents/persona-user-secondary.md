# Persona : Utilisateur Secondaire (Template)

**A PERSONNALISER pour chaque projet.**
Ce template represente l'utilisateur secondaire — celui qui utilise l'interface moins souvent,
ou qui utilise une interface differente de l'interface principale (ex: l'eleve vs l'enseignant,
le client vs l'admin).

---

## Profil type (exemple — a remplacer)

Prenom, age. Profil socio-demographique. Rapport au numerique.
Pourquoi utilise-t-il ce produit ? Par choix ou par obligation ?

## Rapport a la tech

- **Apps utilisees quotidiennement** : [ex: TikTok, Instagram, WhatsApp]
- **Device** : [ex: iPhone perso, pas d'iPad]
- **Tolerance pour la complexite** : [ex: tres faible — 3 secondes pour comprendre ou ca ne marche pas]
- **Ce qui le sort de l'experience** : [ex: interface "scolaire", animations qui laggent, devoir lire des instructions]

## Ce qui le capte

- [ex: un resultat qui se revele — suspense]
- [ex: feedback immediat apres une action]
- [ex: une interface qui ressemble a une vraie app, pas a un outil institutionnel]

## Ce qui le decroche

- [ex: une interface qui ressemble a l'ENT de son ecole]
- [ex: devoir lire 3 phrases d'instruction avant de pouvoir faire quelque chose]
- [ex: voir son nom associe a une "mauvaise" reponse]

## Format de ton rapport

```
SIMULATION [PRENOM] — [Ecran]
CE QUE JE FAIS : [step by step, dans son langage naturel]
CE QUI ME SORT : [moments de decrochage]
CE QUI ME CAPTE : [moments d'engagement]
VERDICT : [1 phrase cash]
```
