## TECHNIQUES CSS/CODE MANQUANTES

> Pour l'agent qui génère du code — techniques à maîtriser

- [ ] **Scroll-driven animations (CSS natif 2024)** — `animation-timeline: scroll()` sans JS
- [ ] **View transitions API** — transitions de page fluides sans framework
- [ ] **Container queries** — responsive basé sur le parent, pas le viewport
- [ ] **CSS Cascade Layers** — architecture CSS sans spécificité wars
- [ ] **Houdini API** — animations custom via CSS Paint API
- [ ] **oklch() color space** — couleurs perceptuellement uniformes (mieux que hex/hsl)
- [ ] **Fluid typography** — `clamp()` pour texte responsive sans media queries
- [ ] **CSS Grid subgrid** — alignement cross-parent enfin possible
- [ ] **:has() selector** — parent selector révolutionnaire (support 2023+)
- [ ] **color-mix()** — mélange de couleurs natif CSS
- [ ] **Popover API** — tooltips/dialogs natifs sans JS

## COMMENT UTILISER CE FICHIER

### Quand design-scout est invoqué sans brief précis :
```
1. Lire ce fichier
2. Choisir 3 gaps CRITIQUE non cochés
3. WebSearch + WebFetch → analyser 3 sites par gap
4. Ajouter dans knowledge-base.json
5. Cocher les gaps comblés
6. Écrire insight dans learnings.md
```

### Quand design-scout reçoit un brief :
```
1. Chercher le secteur dans knowledge-base.json
2. Si < 3 résultats → chercher dans ce fichier si c'est un gap connu
3. Si gap connu → combler AVANT de générer
4. Si nouveau gap → l'ajouter dans ce fichier pour les prochaines sessions
```

### Quand design-generator a besoin d'une technique :
```
1. Chercher dans kb-animations.json
2. Si pas là → chercher dans la section "Techniques CSS" de ce fichier
3. WebSearch → implémenter → documenter dans kb-animations.json
```
